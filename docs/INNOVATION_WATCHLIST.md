# Innovation Watchlist

## Призначення
Сюди потрапляють речі, які ще не варто брати, але треба моніторити регулярно.

---

## Коли додавати сюди
- рішення перспективне, але сире;
- є сильна ідея, але слабка зрілість;
- потрібні додаткові релізи, відгуки або API;
- зараз не пріоритетно, але може різко стати важливим.

## Поточний список

## Managed Agents Platforms
- Категорія: agents / orchestration
- Чому в watchlist: можуть сильно зменшувати інфраструктурний тягар, але не заміняють memory governance, audit і deterministic medical workflows
- Що повинно змінитися, щоб перейти в backlog: стабільний API, кращий контроль, enterprise-grade observability, зрозуміла інтеграція з власною пам'яттю
- Дата додавання: 2026-04-13
- Останній перегляд: 2026-04-13

## Long-term Memory Startups
- Категорія: memory
- Чому в watchlist: сильна ідея, але часто забагато маркетингу і замало зрілої керованості
- Що повинно змінитися, щоб перейти в backlog: явна governance-модель, контроль джерел істини, traceability, практичні integration patterns
- Дата додавання: 2026-04-13
- Останній перегляд: 2026-04-13

## Consumer AI Email Prioritization Tools
- Категорія: inbox / prioritization / reminders
- Чому в watchlist: дуже корисні UX-патерни, але небезпечно переносити їх 1-в-1 у критичні середовища
- Що повинно змінитися, щоб перейти в backlog: чіткий adaptation plan для admin / clinic / medical workflows
- Дата додавання: 2026-04-13
- Останній перегляд: 2026-04-13
