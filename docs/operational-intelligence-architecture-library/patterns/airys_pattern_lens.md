# AIRYS Pattern Lens

This document defines how to judge patterns specifically for AIRYS.

AIRYS is not just another assistant shell.
Its architecture should be evaluated through the lens of long-term trust, privacy, memory discipline, emotional usefulness, and real-life continuity.

## Core priorities for AIRYS

Patterns are more valuable when they strengthen:
- privacy-first architecture
- local-first or user-controlled memory direction
- long-horizon coherence
- emotional stabilization without manipulation
- proactive but bounded assistance
- disciplined context systems
- user agency and reversibility

## High-value pattern areas

### 1. Privacy-preserving memory patterns
Questions:
- does the pattern keep user control clear
- can memory be segmented and deleted cleanly
- does it avoid unnecessary retention
- does it separate canon from ephemeral state

### 2. Long-horizon continuity patterns
Questions:
- can the system stay coherent across weeks and months
- does it avoid stale context accumulation
- can active context expire or be revalidated

### 3. Emotional support boundaries
Questions:
- does the pattern help stabilize without pretending to replace reality
- are interventions bounded and explainable
- does it preserve user autonomy

### 4. Ambient assistance patterns
Questions:
- does proactivity feel helpful rather than intrusive
- are triggers grounded in observable signals
- can the user override or disable behavior easily

### 5. Trust architecture patterns
Questions:
- can the user inspect what the system knows
- can important behavior be explained
- are memory and actions reversible where appropriate

## AIRYS-specific rejection criteria

A pattern is a poor fit if it:
- maximizes retention without discipline
- feels manipulative or dependency-seeking
- hides memory behavior from the user
- relies on vague personality magic instead of architecture
- confuses emotional usefulness with emotional theater

## Design principle

AIRYS should feel deeply helpful, but never invasive.
That requires stronger architecture discipline than most companion products.

## Promotion bias

Prefer patterns that make AIRYS:
- more trustworthy
- more coherent over time
- more private by design
- more supportive without becoming manipulative

That is the path toward a genuinely differentiated companion intelligence system.
