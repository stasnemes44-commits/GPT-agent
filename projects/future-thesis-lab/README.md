# Future Thesis Lab

Це ізольований стратегічний модуль для OpenClaw / GPT-agent.

Його призначення:
- зберігати довгострокову future thesis;
- відокремлювати 90% комерційного execution від 10% frontier research;
- фільтрувати нові великі ставки через явні критерії;
- вести signal log, watchlist, ADR та регулярні reviews;
- не дозволяти стратегічному шуму забруднювати щоденний execution loop.

## Важливий принцип
Це **не** глобальний always-on context для всіх сесій агента.
Це **окремий модуль**, який має використовуватися часто, але вибірково:
- при monthly review;
- при quarterly thesis review;
- при оцінці нових напрямків;
- при frontier research;
- при product-thesis decisions.

## Базова модель використання
- always visible: короткий summary і route rule;
- on-demand: canonical thesis, signal log, pain repository, watchlist, ADR;
- scheduled: monthly / quarterly reviews.

## Поточний статус
Цей модуль був доданий в репозиторій як ізольований стратегічний контур.
Він не претендує на опис усієї поточної архітектури OpenClaw.
Його завдання — дати чисту основу для подальшого архітектурного аудиту та інтеграції.

## Головна future thesis
Поточна довга ставка:
**AI Health Equity OS** — приватний, мультимодальний, керований правилами агентний контур, який допомагає людям проходити шлях до правильної допомоги з меншим хаосом, меншою затримкою і меншим стражданням.

## Структура модуля
- `AGENTS.md` — правила використання модуля агентом
- `context/` — canonical thesis і короткий summary
- `signals/` — журнал сигналів
- `pain_repository/` — карта людського болю
- `watchlists/` — активні та відкладені ставки
- `decisions/` — ADR та рішення
- `reviews/` — monthly / quarterly reviews
- `roadmaps/` — короткий операційний горизонт по 10% frontier track

## Інтеграційне правило
Не підключати цей модуль повністю у щоденний глобальний контекст.
Підключати або короткий summary, або окремі файли за тригерами.
